Project 3 - SQL Data Analysis
Overview
This project was completed as part of the DecodeLabs Data Analytics Internship Program (Project 3) 
This project focuses on using SQL queries to extract meaningful insights from an e-commerce dataset. The objective is to analyze business data using SQL operations such as filtering, sorting, grouping, and aggregation.

Objectives
Perform data retrieval using SQL queries
Apply filtering conditions using WHERE
Sort data using ORDER BY
Group records using GROUP BY
Perform aggregations using COUNT, SUM, and AVG
Generate business insights from transactional data
Tools Used
Python
SQLite
Pandas
Jupyter Notebook
Dataset Information
Dataset: Cleaned E-commerce Orders Dataset
Total Records: 1200
Source: Project 1 (Data Cleaning)
SQL Operations Performed
1. SELECT Query

Retrieved records from the orders table.

2. WHERE Clause

Filtered records based on specific conditions such as payment method and product type.

3. ORDER BY Clause

Sorted records based on TotalPrice in descending order.

4. COUNT Function

Calculated the total number of orders.

5. AVG Function

Calculated the average order value.

6. SUM Function

Calculated the total revenue generated.

7. GROUP BY Product

Analyzed order counts for each product category.

8. GROUP BY Payment Method

Compared customer payment preferences.

9. GROUP BY Referral Source

Analyzed customer acquisition channels.

Key Findings
Total Orders: 1200
Average Order Value: ₹1053.97
Online is the most preferred payment method.
Printer is the most frequently ordered product.
Instagram is the leading referral source.
SQL aggregations helped summarize business performance effectively.
Conclusion

SQL was successfully used to query, filter, sort, group, and aggregate business data. The analysis provided valuable insights into customer behavior, product demand, payment preferences, and referral source performance. This project demonstrates fundamental SQL skills required for data analysis and business reporting.
